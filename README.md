####  项目名称： 驾信 

###### APP下载地址：
[http://www.car2hub.com/ios/jx/](http://www.car2hub.com/ios/jx/)

###### 主要功能描述：
该项目为一个集合的对车进行控制、聊天和导航为一体的应用。

######  主要工作内容: 

* 完成整个项目的架构，用三层架构把界面层、服务层和网络层比较彻底的区分开来，大大的降低大模块之间的耦合度，有关网络数据之间的逻辑处理都放到服务层上，网络层管理网络请求，界面层负责界面显示，界面层请求数据唯一要做的就是组织参数，发起请求，显示结果就可以，不用做多余的处理。
* 对各个界面进行整理、抽离，分化出各个工具类、助手类、可重用类并一一完成。
* 完成HTTP网络库的封装，这个库一共实现的功能有：网络请求队列、全程唯一的数据线程用于请求网络、多点回调、界面友好的分页请求、网络请求的数据自动赋值等功能。
* 实现socket的封装，界面层简单调用就能发出命令对车进行控制。
* 用XMPP框架实现聊天通信系统。
* 在会议上为后台服务器同事提出双管道缓存池方案并解说具体实现思路为后台解决的token越来越多的问题。
* 对公司产品的功能提出建设性的意见。
* 对公司内部存在的问题向上级反馈并提示建议性的解决方案。

###### 最困难的问题：
在驾信一期的项目项目里，网络库是把请求到的数据都写进数据库里，然后界面从数据库里取数据，这样效率不高且麻烦，并且因为一期的项目耗时半年之久和经过的不同的人手里，导致的很多问题，特别是代码沉余太多并且因为有不同版本的等严重的适配问题。
###### 解决方案：
在一边完成一期项目需求的同时利用晚上和周未的时间对整个项目进行重写，定为驾信二期；在重写之前对整个项目的界面进行抽离和分化出各个可重用类，平均每个viewController的代码减少的50%；因为我们的项目更多功能里的各个界面基本都是一致了，我就把更多里的所有界面的界面逻辑都抽离到一个父类里头,而子类 只要设置好数据源，这样做之后，更多里有一个界面的代码由800多行减少到150行。

   在完成整个项目的90%的时候，告诉经理并且完全替代的一期的工程；经理观看的之后对此大力赞许；并且经过测试在不同版本不同尺寸之间的适配不用多写一行代码(包括iphone6和iphone6+)。
